## Objective
Define the **partner-ready developer preview contract**: how SwapGraph accepts intents and returns proposals via REST and webhooks.

This milestone is contract-first and fixtures-first so the agent can verify it deterministically.

## Scope
- REST endpoint manifest (`docs/spec/api/manifest.v1.json`).
- API contract doc (`docs/spec/API.md`).
- API wrapper schemas (request/response objects) under `docs/spec/schemas/`.
- API examples under `docs/spec/examples/api/`.
- Verifier that checks:
  - manifest well-formed,
  - referenced schemas exist,
  - all API examples validate against schemas,
  - emits stable proof artifacts.

## Non-goals
- No real HTTP server.
- No real auth implementation yet (we only specify contract).
- No matching engine correctness yet (proposals can be fixture-provided at this stage).

## Acceptance criteria
- `npm run verify:m2` exits 0.
- `node verify/runner.mjs milestones/M2.yaml` exits 0.
- Proof artifacts exist at `artifacts/milestones/M2/latest/*`.

## Required artifacts
- `docs/spec/API.md`
- `docs/spec/api/manifest.v1.json`
- `docs/spec/examples/api/_manifest.v1.json`
- `artifacts/milestones/M2/latest/api_contract_validation.json`
- `artifacts/milestones/M2/latest/commands.log`
