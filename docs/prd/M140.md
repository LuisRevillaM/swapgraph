# M140 — Marketplace listings/edges/feed contract implementation (fixtures-first)

## Objective
Implement the first executable Marketplace vNext API slice from the self-contained spec:
- market listings lifecycle,
- market edges lifecycle,
- market feed projection,
with deterministic contracts, reason codes, and verifier artifacts.

## Scope
1. Runtime market service and route wiring:
   - `POST /market/listings`
   - `PATCH /market/listings/{listing_id}`
   - `POST /market/listings/{listing_id}/pause`
   - `POST /market/listings/{listing_id}/close`
   - `GET /market/listings/{listing_id}`
   - `GET /market/listings`
   - `POST /market/edges`
   - `POST /market/edges/{edge_id}/accept`
   - `POST /market/edges/{edge_id}/decline`
   - `POST /market/edges/{edge_id}/withdraw`
   - `GET /market/edges/{edge_id}`
   - `GET /market/edges`
   - `GET /market/feed`
2. Schema + manifest + examples for all above contracts.
3. Auth scope extension:
   - add `market:read`, `market:write` and validate via auth-contract checker.
4. Deterministic reason-code floor for listing/edge/feed errors.
5. Milestone verifier scenario with deterministic hash-locked output.

## Deterministic reason-code floor
- `market_listing_invalid`
- `market_listing_kind_invalid`
- `market_listing_status_invalid`
- `market_listing_not_found`
- `market_listing_forbidden`
- `market_edge_invalid`
- `market_edge_not_found`
- `market_edge_target_owner_required`
- `market_edge_status_transition_invalid`
- `market_feed_query_invalid`
- `market_cursor_not_found`

## Non-goals (M140)
- No deal/thread/payment-proof/execution-grant contracts yet.
- No matching bridge changes to settlement yet.
- No UI/client implementation.

## Acceptance criteria
M140 is accepted when:
1. all market listing/edge/feed endpoints are runtime-wired and manifest-declared,
2. schema + example contracts validate,
3. scope validator accepts `market:*` and rejects malformed auth declarations,
4. deterministic scenario verifies lifecycle + reason-code behavior and hash-locks output,
5. milestone verifier (`npm run verify:m140` + `node verify/runner.mjs milestones/M140.yaml`) passes.
