# M25 — Settlement write endpoint contracts (schemas + examples + verifier)

## Objective
Add explicit **REST write endpoint contracts** for settlement actions that we already proved via store-backed services (M21–M24).

This milestone is **contract-first**:
- request/response JSON Schemas
- API manifest updates
- examples validated by `scripts/validate-api-contract.mjs`
- a fixtures scenario that proves our API-level wrappers return schema-valid bodies (success + error)

## Endpoints (v1)
Add these endpoints to `docs/spec/api/manifest.v1.json`:
- `POST /settlement/{cycle_id}/start` (`settlement.start`)
- `POST /settlement/{cycle_id}/deposit-confirmed` (`settlement.deposit_confirmed`)
- `POST /settlement/{cycle_id}/begin-execution` (`settlement.begin_execution`)
- `POST /settlement/{cycle_id}/complete` (`settlement.complete`)
- `POST /settlement/{cycle_id}/expire-deposit-window` (`settlement.expire_deposit_window`)

## Response conventions
- Success responses include `correlation_id` in the JSON body (fixtures-first stand-in for `X-Correlation-Id`).
- Error responses use the standard `ErrorResponse` envelope.

- `npm run verify:m25` exits 0.
- `node verify/runner.ts milestones/M25.yaml` exits 0.

- Updated API manifest + examples manifest
- New request/response schemas + examples
