# M142 — Marketplace Deals + Negotiation Threads

Goal: turn accepted market edges into participant-scoped deals with deterministic thread linkage and schema-validated read/write surfaces.

Delivered:
- `POST /market/deals/from-edge/{edge_id}`
- `GET /market/deals/{deal_id}`
- thread auto-linking from deal creation
- feed/listing/thread schema extensions required by deal lifecycle
- `scripts/run-m142-market-deals-threads-scenario.mjs`

Gate:
- accepted edge produces one active deal
- deal state is participant-scoped
- negotiation thread contains both system and participant messages
