# M141 — Marketplace threads/messages contract implementation (fixtures-first)

## Objective
Implement the negotiation conversation contract slice for Marketplace vNext:
- thread creation and read/list,
- thread message append and read/list,
with deterministic auth, idempotency, query behavior, and verifier artifacts.

## Scope
1. Runtime endpoints:
   - `POST /market/threads`
   - `GET /market/threads/{thread_id}`
   - `GET /market/threads`
   - `POST /market/threads/{thread_id}/messages`
   - `GET /market/threads/{thread_id}/messages`
2. State-store support for `market_threads` and `market_messages` namespaces.
3. API manifest + schema + examples for all thread/message contracts.
4. Deterministic verifier scenario for lifecycle, participant authorization, and query/cursor error paths.

## Deterministic reason-code floor
- `market_thread_invalid`
- `market_thread_not_found`
- `market_thread_forbidden`
- `market_message_invalid`
- `market_feed_query_invalid`
- `market_cursor_not_found`

## Non-goals (M141)
- No deal/acceptance settlement bridge contracts.
- No payment proof / execution grant contracts.
- No UI implementation.

## Acceptance criteria
M141 is accepted when:
1. thread/message endpoints are runtime wired and manifest declared,
2. schema + example contracts validate,
3. participant authz is deterministic (non-participants are denied),
4. idempotent writes replay correctly,
5. verifier hash-lock scenario passes and milestone runner returns `overall=true`.
